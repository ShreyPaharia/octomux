# Neovim + LazyVim Setup for octomux-agents

## Problem

Agents and developers working in octomux-agents worktrees need a productive editor
setup. Currently there is no editor configuration in the repo. LazyVim provides a
batteries-included neovim config with LSP, treesitter, telescope, and more вЂ” all
with sensible defaults.

## Solution

Add a repo-local LazyVim configuration under `.config/nvim/` and two scripts:
one to bootstrap the environment, one to launch nvim with the repo config.

All XDG directories (config, data, state, cache) are scoped to the repo's
`.local/` directory so the setup is fully isolated from the user's personal
neovim installation.

## Environment Isolation

Both scripts set all four XDG variables to keep everything repo-local:

```bash
REPO_ROOT="$(cd "$(dirname "$0")/.." && pwd)"
export XDG_CONFIG_HOME="$REPO_ROOT/.config"
export XDG_DATA_HOME="$REPO_ROOT/.local/share"
export XDG_STATE_HOME="$REPO_ROOT/.local/state"
export XDG_CACHE_HOME="$REPO_ROOT/.local/cache"
```

This ensures:
- Config reads from `.config/nvim/` (committed)
- Plugins clone to `.local/share/nvim/lazy/` (gitignored)
- State and cache go to `.local/state/` and `.local/cache/` (gitignored)
- No collision with user's existing `~/.config/nvim` or `~/.local/share/nvim`

## Config Files

### init.lua

Standard LazyVim starter `init.lua` вЂ” requires `config.lazy` to bootstrap
the plugin manager.

### lua/config/lazy.lua

Bootstraps `lazy.nvim` from GitHub if not present, then loads LazyVim as the
base config. This is the standard LazyVim starter pattern:

- Clone `lazy.nvim` to `vim.fn.stdpath("data") .. "/lazy/lazy.nvim"` on first run
  (resolves to `.local/share/nvim/lazy/lazy.nvim` due to XDG_DATA_HOME)
- Configure `lazy.nvim` with the LazyVim plugin spec
- Import from `lua/plugins/` for additional plugin specs

### lua/plugins/extras.lua

Enables three LazyVim extras for this TypeScript/React codebase:

```lua
return {
  { import = "lazyvim.plugins.extras.lang.typescript" },
  { import = "lazyvim.plugins.extras.lang.tailwind" },
  { import = "lazyvim.plugins.extras.linting.eslint" },
}
```

These provide: ts_ls/vtsls, tsx treesitter grammars, tailwindcss LSP with
completions, and eslint LSP integration.

### stylua.toml

Standard Lua formatter config from the LazyVim starter template.

## Scripts

### scripts/setup-nvim.sh

Idempotent bootstrap script:

1. Check if `nvim` is on PATH
2. If missing and on macOS, run `brew install neovim`
3. If missing and not macOS, print error with install instructions and exit 1
4. Verify neovim version >= 0.10.0 (required by current LazyVim)
5. Set all four XDG variables to repo-local paths
6. Run `nvim --headless "+Lazy! sync" +qa` to pre-install all plugins
7. Note: treesitter grammars may compile on first interactive launch

### scripts/nvim.sh

Wrapper script that sets all XDG variables to the repo's directories,
then execs `nvim` with all passed arguments. Agents and users run
`./scripts/nvim.sh <file>` to use the repo-local config.

## Git Considerations

- `.config/nvim/` Lua files, `stylua.toml`, and `lazy-lock.json` are committed
- `.local/` is added to `.gitignore` (plugin data, state, cache)
- `.config/nvim/.luarc.json` is added to `.gitignore` (generated by lua-language-server)

## Non-Goals

- No custom keymaps, autocmds, or options beyond LazyVim defaults
- No Nix/declarative package management
- No support for non-macOS auto-install (manual install instructions only)
- No integration with existing npm scripts or package.json
